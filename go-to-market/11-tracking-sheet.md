# DAILY TRACKING — Pipeline Math

**Hard rule:** update this every day at 5pm. No exceptions. No "I'll catch up Friday."
**Why:** at $0 ARR ur only feedback loop is leading indicators. Lagging metrics (revenue) take 30+ days. U need to know within a week if outreach is broken.

## RED FLAGS — Stop & Diagnose

| Symptom | Diagnosis | Fix |
|---|---|---|
| <2% reply rate after 500 sends | Subject line / list quality / domain warmth | A/B subject lines, audit list, check Glockapps |
| >5% bounce rate | Bad list / domain not warm | Pause campaign, re-warm, switch enrichment provider |
| Replies but no demo books | Reply is "not interested" rather than engaged | Email body too pushy; soften CTA, ask qualifying ?'s |
| Demos booked but no shows | No reminder cadence | Add Cal.com reminder, 24h SMS, day-of SMS |
| Demos but no closes | Pricing or trust | Test downsell to $179, add 30-day refund, pull case study earlier |
| Closes but immediate churn | Onboarding gap | Day-1 success call, day-7 check-in, day-30 ROI report |

## WHERE TO LOG

Pick one. Don't pick three.

- **Notion:** simplest, free, mobile. Recommended.
- **Airtable:** if u want pipeline kanban view + automations.
- **Supabase:** if u want this in the same db as customer data (good for unified analytics later).
- **Spreadsheet:** Google Sheet. Do NOT use Excel — u'll lose it.

**Set it up Sunday night before Monday starts. 30 min max. Don't perfect it. Tracking that exists > tracking that's beautiful.**

---

## ONE METRIC THAT MATTERS

If u look at one number daily, it's: **demos completed this week.**

- Leads, emails, calls = inputs
- Replies, opens = leading indicators
- Customers closed = lagging
- **Demos completed = the bridge**

5 demos/wk → 1 close/wk → $25K ARR/mo run rate
10 demos/wk → 2 closes/wk → $50K ARR/mo
20 demos/wk → 4 closes/wk → $100K ARR/mo

The whole game in 12 months is getting from 5 demos/wk by Aug to 20 demos/wk by Apr.
